> 时间：2026-03-05  
> 目标：打造智能书籍解析系统，为书籍营销提供精准洞察

---

## 🎯 核心问题

### 我们要解决什么问题？
1. **书籍营销盲目**：不知道目标读者是谁，广告投放浪费
2. **素材制作低效**：人工提取卖点耗时，质量不稳定
3. **人群定位模糊**：不清楚哪些人会买，哪些人不会买
4. **转化率低**：广告文案千篇一律，无法精准击中痛点

### 我们的解决方案是什么？
**AI 驱动的书籍智能解析系统**
- 输入：书籍 ID + 前 10-30 章内容
- 输出：完整的内容分析 + 人群画像 + 营销素材 + 投放策略

## 🧠 深度思考

### 问题 1：如何判断一本书的「爆款潜力」？

**指标体系**：
```
爆款指数 = 
  内容质量 (40%) +
  市场需求 (30%) +
  营销潜力 (20%) +
  时机匹配 (10%)
```

**内容质量**：
- 人物立体度（主角是否有成长弧线）
- 情节紧凑度（是否有拖沓章节）
- 情感浓度（是否能引发共鸣）
- 爽点密度（是否有足够的爽点）

**市场需求**：
- 题材热度（当前市场是否流行该题材）
- 竞品分析（同类书籍的表现）
- 搜索指数（相关关键词的搜索量）

**营销潜力**：
- 高光片段数量（适合做短视频的片段）
- 金句数量（适合做海报的金句）
- 话题性（是否能引发讨论）

**时机匹配**：
- 是否有热点事件可以蹭
- 是否有节日可以结合（情人节推爱情小说）

### 问题 2：如何避免「推荐不准」？

**负面案例识别**：
- 不要给失恋的人推虐恋小说
- 不要给职场新人推宫斗小说
- 不要给中年人推校园小说

**A/B 测试**：
- 同一本书，不同文案，测试哪个转化率高
- 同一人群，不同书籍，测试哪个更受欢迎

**用户反馈闭环**：
- 收集用户评价（好评/差评）
- 分析弃书原因
- 优化推荐算法

### 问题 3：如何提高「转化率」？

**漏斗优化**：
```
曝光 → 点击 → 试读 → 付费 → 完读
```

**每个环节的优化点**：
- **曝光**：精准投放，避免浪费
- **点击**：标题党（但不能过度）、封面吸睛
- **试读**：前 3 章必须抓人，设置悬念
- **付费**：在高潮前设置付费墙
- **完读**：持续更新，保持热度

```
   - 1: 极度正面（喜悦、兴奋、高潮）

2. 节奏 BPM（像心跳一样）
   - 60: 慢节奏（平静、铺垫）
   - 90: 中等节奏
   - 120+: 快节奏（冲突、高潮）

3. 爽点密度（0-1）
   - 0: 无爽点
   - 0.5: 中等
   - 1: 爽点密集

4. 冲突强度（0-1）
   - 0: 无冲突
   - 0.5: 中等冲突
   - 1: 强烈冲突

【输出格式】
严格按照 JSON 格式输出。
```

### 2. 上头指数预测（Addiction Index）

**概念**：预测读者会不会"上头"（沉迷追读），像游戏的成瘾机制分析

#### 评分体系
```json
{
  "addictionIndex": {
    "score": 8.7,
    "level": "极易上头",
    
    "factors": {
      "suspenseHooks": {

        "score": 8.5,
        "weight": 0.20,

      },

    ]
  }

```

#### 计算公式
```
上头指数 = 
  悬念钩子 × 0.25 +
  情感投入 × 0.20 +
  节奏控制 × 0.15 +
  奖励频率 × 0.20 +
  章末悬念 × 0.20
```

#### 应用场景
- **平台运营**：预测哪些书会火，重点推广
- **作者优化**：找到"不上头"的原因，针对性改进
- **付费策略**：在"最上头"的时候设置付费墙
- **营销文案**：突出"上头点"吸引读者

#### AI Prompt 模板
```
你是一位资深的用户心理学家和内容分析师。请分析以下书籍，预测"上头指数"：

【书籍信息】
书名：{{bookTitle}}
章节内容：{{chapters}}

【分析维度】
1. 悬念钩子（0-10 分）
   - 有多少个未解之谜？
   - 悬念设置是否巧妙？
   - 是否让读者欲罢不能？

2. 情感投入（0-10 分）
   - 读者是否会对主角产生共鸣？
   - 是否会为主角的遭遇感到愤怒/心疼？
   - 是否会代入主角的视角？

3. 节奏控制（0-10 分）
   - 节奏是否张弛有度？
   - 是否有拖沓的章节？
   - 高潮是否来得及时？

4. 奖励频率（0-10 分）
   - 多久给读者一个"爽点"？
   - 是否持续给予正反馈？
   - 是否让读者有"再看一章"的冲动？

5. 章末悬念（0-10 分）
   - 每章结尾是否有悬念？
   - 是否让读者想立即看下一章？
   - 悬念设置是否自然？

【预测任务】
1. 计算上头指数（0-10 分）
2. 预测一口气看完的概率
3. 预测平均阅读时间
4. 预测弃书率
5. 找出"不上头"的风险点

【输出格式】
严格按照 JSON 格式输出。
```
